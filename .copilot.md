# Copilot Configuration

Project configuration and guidelines for AI-assisted development.

## Project Information

- **Name**: Symfony Console Getting Started
- **Type**: PHP Console Application
- **Framework**: Symfony Console Component
- **Language**: PHP
- **Package Manager**: Composer

## Development Guidelines

### Code Style
- Follow PSR-12 coding standards
- Use type hints for all function parameters and return types
- Keep classes and methods focused and single-purpose

### Testing
- Write tests for new features
- Ensure all tests pass before committing
- Use PHPUnit for unit testing

### Documentation
- Add docblocks to all public methods
- Update README.md when adding new commands
- Document command usage in the command class itself

### Commits
- Write clear, descriptive commit messages
- Reference issues when applicable
- Use conventional commit format when possible

## Key Files

- `composer.json` - Project dependencies and configuration
- `composer.lock` - Locked dependency versions
- `bin/` - Console application entry points (if present)
- `src/` - Source code (if present)

## Common Commands

```bash
# Install dependencies
composer install

# Update dependencies
composer update

# Run any defined scripts in composer.json
composer run-script <script-name>
```

## Before Making Changes

1. Check existing code patterns in the project
2. Run tests to ensure baseline functionality
3. Review AGENTS.md for appropriate agent to use
4. Create a plan for larger changes
